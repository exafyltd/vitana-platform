## What it is

Recall is the Recall screen in the Memory module. It gives the user a focused place to understand the current state, choose the next step, and ask Vitana for help.

Memory lets Vitana learn from diary entries, timeline events, recalled facts, and permission boundaries.

## Why it matters

Memory lets Vitana learn from diary entries, timeline events, recalled facts, and permission boundaries. It should help the user live better through the five longevity pillars while making useful community or income opportunities easier to act on.

## Where to find it

Direct URL: `/memory/recall`.

Sidebar path: **Memory → Recall**.

Module: **Memory**.

## What you see on this screen

- Diary, timeline, recall, or permission entries
- Source, date, category, and sensitivity labels
- Add, edit, forget, save, pin, or export controls
- Links to recommendations that use memory

## How to use it

1. Use Diary for today, Timeline for history, Recall for what Vitana remembers, and Permissions for boundaries.
2. Inspect source before trusting a recalled fact.
3. Correct or forget wrong memories.
4. Save important context that should improve future guidance.

## What you can ask Vitana

- "Open my Daily Diary."
- "What do you remember?"
- "Forget this."
- "Add this life event."

## Guidance for Vitana Assistant

Memory lets Vitana learn from diary entries, timeline events, recalled facts, and permission boundaries. Explain the screen in plain language first, then offer the next safe action. Ask for confirmation before sending messages, making purchases, booking services, changing privacy, sharing data, altering subscriptions, or deleting anything.
