## What it is

Privacy, Data Monetization, and VAEA describe how Vitana uses data with user control. VAEA stands for Vitana Autonomous Economic Actor and is an internal name for autonomous economic behavior such as matching, referrals, opportunity discovery, and monetization workflows. In user-facing language, explain the capability as Autopilot Referrals, Data Monetization, or income Autopilot depending on context.

The user can choose whether profile, activity, marketplace, content, or health-adjacent data may be used for recommendations, matching, campaigns, or monetization. Sensitive health information requires special care and should never be shared casually.

## Why it matters

The longevity economy only works if users trust it. Data can create value by improving matches, finding clients, personalizing services, and surfacing opportunities, but that value must be permissioned, auditable, and reversible.

Vitana should always be able to answer: what data is used, why it is used, who can see it, what benefit it creates, and how to turn it off.

## Where to find it

Use Settings -> Privacy for account-level privacy. Use Sharing -> Data & Consent for campaign, data sharing, and monetization permissions. Use Wallet -> Rewards to see value created. Use Memory Permissions to control what the assistant remembers.

## How to use it

1. Explain the benefit and risk before enabling any monetization workflow.
2. Let the user choose specific categories rather than a single broad consent.
3. Keep health data private unless the user explicitly approves a specific use.
4. Show the audit history when the user asks what was shared.
5. Revoke, pause, export, or delete data on request, following app policy.

## What you can ask Vitana

- "What data are you using?"
- "Turn on data monetization."
- "Show my consent history."
- "Stop sharing my profile data."
- "How does Autopilot earn for me?"
